# PlanPro-UX-Lifecycle-Map

PlanPro is a web based scheduling platform designed for small businesses. The system allows users to manage appointments, employee schedules and customer notifications. The company wants to reduce onboarding friction, improve task efficiency, lower support requests and encourage long term adoption. 
To achieve these goals, the UX activities should be integrated throughout the entire product lifecycle instead of only being used at the end of the development. One important concept in UX is called shifting left, which means involving UX activities as early in the project as possible instead of waiting until the system is finished. 
The activities involve user interviews, task analysis and persona development in order to help the team understand users before design and the development begins and by identifying users needs as early as possible it makes the team avoid creating features that do not provide value.
By integrating UX early it also helps reduce long term development cost. The problems that are discovered after the coding is completed is usually more expensive to fix because it requires redesigning features and repeating testing activities.
By using wireframes, prototypes and usability evaluations early in the proves, issues can be corrected before they become larger problems, which helps reduce rework as well as it saves time and company resources. 
The lifecycle integration also supports the measurable business outcomes and a better onboarding experience helps new business owners learn the system faster. The improved workflows also help employees complete their tasks more efficiently. 
In addition, an easy to use system can also reduce support requests because users are less likely to become confused. These improvements keep customer satisfaction and makes them more likely to continue to use PlanPro overtime.
The UX map for PlanPro shows that planning ahead is better than waiting until the end to test the system. Instead of waiting until the release phase to evaluate usability, the UX activities should be included in the discovery, design, release and post release phases, since each phase helps to improve the quality of the product in a different way.      
The discovery phase is used to learn about the users and what they need. The design phase focuses on creating and testing ideas before the coding starts. The development phase makes sure that features are built correctly and will meet the needs of the users. 
During the release phase, the finished product is tested before it is fully launched. In the post release phase the team will collect feedback from users and make improvements based on what they have learned. 
By using UX throughout the entire lifecycle it helps create a product that is easier for people to use, so instead of waiting for customers to report any problems the PlanPro team can find and fix any issues early on. This will improve the product quality, reduce costs and support long term business success.
